# TubeMixer 要件定義書

## Context
YouTubeでチャンネル登録しているチャンネルの動画を、グループ化・日付フィルタで絞り込み、その結果からYouTube再生リストを作成して視聴したいという個人ニーズに応えるWebアプリケーション。

---

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| プロダクト名 | TubeMixer |
| 種別 | Webアプリケーション（PWA対応） |
| 利用者 | 個人専用（自分のみ） |
| 対象デバイス | iPhone（メイン）、PC（サブ） |

| レイヤー | 技術 |
|----------|------|

## 3. 機能要件

### 3.1 認証機能
- Googleアカウントでログイン（OAuth 2.0）
- YouTube Data API へのアクセス権限を取得
- 必要なスコープ:
  - `youtube.readonly`（チャンネル登録・動画情報の取得）
  - `youtube`（再生リストの作成・動画追加）

### 3.2 チャンネル登録一覧の取得
- ログインユーザーのチャンネル登録一覧をYouTube APIから取得
- チャンネル名・アイコン・チャンネルIDを表示
- **API節約**: 取得結果をキャッシュし、手動リフレッシュボタンで更新

### 3.3 チャンネルグループ管理
- **グループの作成**: グループ名を付けて新規作成
- **グループの編集**: グループ名の変更、グループの削除
- **チャンネルの割り当て**:
  - チャンネル登録一覧からグループに追加・削除
  - **チャンネル名でのフィルタリング検索**（インクリメンタルサーチ）
  - 1つのチャンネルは1つのグループにのみ所属（排他的）
- **グループ設定の永続化**: サーバーサイドまたはローカルストレージに保存

### 3.4 動画フィルタリング
- **チャンネルグループ選択**: 作成したグループを選択
- **日付範囲指定**: 開始日〜終了日で投稿日をフィルタ
- 選択グループに属する全チャンネルの動画を取得し、日付でフィルタ
- **API節約**: 取得済み動画情報のキャッシュ

### 3.5 フィルタ結果の表示
- フィルタ結果の動画一覧をリスト表示
  - サムネイル、タイトル、チャンネル名、投稿日、再生時間
- 動画の選択/解除（再生リストに含める動画を選べる）
- 全選択/全解除ボタン

### 3.6 YouTube再生リスト作成
- フィルタ結果（または選択した動画）からYouTube上に再生リストを作成
- 再生リスト名は自動生成（例: `TubeMixer: {グループ名} {日付範囲}`）またはユーザー指定
- 作成後、再生リストのURLを表示
- **YouTubeアプリで開く**:
  - `youtube://` スキームでYouTubeアプリを起動（インストール済みの場合）
  - フォールバックとして `https://www.youtube.com/playlist?list=...` をブラウザで開く
- 再生リストの公開設定: **非公開**（デフォルト）

## 4. 画面構成

### 4.1 ログイン画面
- Googleログインボタン

### 4.2 メイン画面（ダッシュボード）
- グループ一覧の表示
- グループ選択 → 日付範囲指定 → フィルタ実行のフロー

### 4.3 グループ管理画面
- グループの作成・編集・削除
- チャンネル割り当て（検索付き）

### 4.4 フィルタ結果画面
- 動画一覧（選択可能）
- 「再生リストを作成」ボタン
- 作成完了後のリンク表示

## 5. 非機能要件

### 5.1 API クォータ管理
- YouTube Data API の無料枠: **1日 10,000 ユニット**
- 主なコスト:
  - チャンネル登録一覧取得: 1ユニット/リクエスト
  - 動画検索(search.list): **100ユニット/リクエスト** ← 最もコストが高い
  - 動画詳細取得(videos.list): 1ユニット/リクエスト
  - 再生リスト作成: 50ユニット/リクエスト
  - 再生リストへの動画追加: 50ユニット/リクエスト
- **節約戦略**:
  - search.list の代わりに playlistItems.list（チャンネルのアップロード動画一覧、1ユニット）を使用
  - 取得結果をセッション内でキャッシュ
  - API使用量の目安を画面に表示

### 5.2 レスポンシブデザイン
- iPhone（モバイルファースト）で快適に操作できるUI
- PC でも利用可能

### 5.3 PWA対応
- ホーム画面に追加可能
- オフライン時はキャッシュ済みデータを表示

## 6. 制約事項・注意点

- YouTube Data API の利用には Google Cloud Console でプロジェクト作成・API有効化が必要
- OAuth同意画面の設定が必要（個人利用なら「テスト」モードで運用可能）
- 再生リストへの動画追加は **1本あたり50ユニット** 消費するため、大量の動画追加時はクォータに注意
  - 例: 50本追加 = 2,500ユニット + 再生リスト作成50 = **2,550ユニット**
- Vercel無料枠の制限: サーバーレス関数の実行時間10秒（動画大量取得時に考慮が必要）

## 実装方針

### ディレクトリ構成（概要）
```
TubeMixer/
├── src/
│   ├── app/              # Next.js App Router
│   │   ├── page.tsx      # ダッシュボード
│   │   ├── login/        # ログイン画面
│   │   ├── groups/       # グループ管理画面
│   │   └── api/          # API Routes
│   │       ├── auth/     # NextAuth.js
│   │       └── youtube/  # YouTube API プロキシ
│   ├── components/       # UIコンポーネント
│   ├── lib/              # ユーティリティ
│   │   ├── youtube.ts    # YouTube API クライアント
│   │   └── cache.ts      # キャッシュ管理
│   └── types/            # 型定義
├── public/               # 静的ファイル・PWAアセット
├── next.config.js
├── tailwind.config.ts
├── tsconfig.json
└── package.json
```

### 検証方法
1. `npm run dev` でローカル起動し、Googleログイン → チャンネル一覧取得を確認
2. グループ作成 → チャンネル割り当て → フィルタ実行 → 動画一覧表示を確認
3. 再生リスト作成 → YouTube上に再生リストが作成されることを確認
4. iPhoneのSafariからアクセスし、モバイルUIとYouTubeアプリ連携を確認
5. Vercelにデプロイし、本番環境で動作確認
