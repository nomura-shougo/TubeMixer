# TubeMixer 要件定義書

## Context
YouTubeでチャンネル登録しているチャンネルの動画を、グループ化・日付フィルタで絞り込み、その結果からYouTube再生用URLを生成して視聴したいという個人ニーズに応えるWebアプリケーション。

---

## 1. プロダクト概要

| 項目 | 内容 |
|------|------|
| プロダクト名 | TubeMixer |
| 種別 | Webアプリケーション（最低限のPWA対応） |
| 利用者 | 個人専用（自分のみ） |
| 対象デバイス | iPhone（メイン）、PC（サブ） |

## 2. 技術スタック

| レイヤー | 技術 |
|----------|------|
| フレームワーク | Next.js (App Router) |
| 言語 | TypeScript |
| スタイリング | Tailwind CSS |
| 認証 | NextAuth.js + Google OAuth 2.0 |

| ホスティング | Vercel（無料枠） |
| API | YouTube Data API v3 |

## 3. 機能要件

### 3.1 認証機能
- Googleアカウントでログイン（OAuth 2.0）
- YouTube Data API へのアクセス権限を取得
- 必要なスコープ:
  - `youtube.readonly`（チャンネル登録・動画情報の取得）
- OAuth同意画面は「テスト」モードで運用（リフレッシュトークンの有効期限が7日間のため、7日ごとに再ログインが必要）

### 3.2 チャンネル登録一覧の取得
- ログインユーザーのチャンネル登録一覧をYouTube APIから取得
- チャンネル名・アイコン・チャンネルIDを表示
- 取得時に各チャンネルのアップロード再生リストID（`contentDetails.relatedPlaylists.uploads`）もまとめて取得し、メモリ上に保持（`channels.list` で最大50件一括取得可能）
- キャッシュは行わず、毎回APIから取得する（クォータ的に問題なし）

### 3.3 チャンネルグループ管理
- **グループの作成**: グループ名を付けて新規作成
- **グループの編集**: グループ名の変更、グループの削除
- **チャンネルの割り当て**:
  - チャンネル登録一覧からグループに追加・削除
  - **チャンネル名でのフィルタリング検索**（インクリメンタルサーチ）
  - 1つのチャンネルは複数のグループに所属可能
- **グループ設定の永続化**: Vercel KV に保存（キー: `user:{googleId}:groups`、ログアウト時も保持、デバイス間同期）

### 3.4 動画フィルタリング
- **チャンネルグループ選択**: 作成したグループを選択
- **日付範囲指定**: 開始日〜終了日で投稿日をフィルタ（想定: 3日〜1週間程度）
- 選択グループに属する全チャンネルの動画を取得し、日付でフィルタ
  - 想定規模: 1グループあたり20〜30チャンネル
  - 投稿頻度が高いチャンネル（1日数本以上）は考慮外
- キャッシュは行わず、毎回APIから取得する
- **取得方式**: サーバーサイド（API Routes）経由で取得し、フロントエンドから3〜5件ずつ並列リクエスト + 進捗表示（完了チャンネル数/全チャンネル数）

### 3.5 フィルタ結果の表示
- フィルタ結果の動画一覧をリスト表示
  - サムネイル、タイトル、チャンネル名、投稿日
- **表示順**: 投稿日の新しい順（全チャンネル混合）
- 動画の選択/解除（再生用URLに含める動画を選べる）
- 全選択/全解除ボタン

### 3.6 再生用URL生成
- フィルタ結果（または選択した動画）から `watch_videos` URL を生成
  - APIクォータ消費なし（非公式URL）
  - 動画数が約50本を超える場合は警告メッセージを表示（URLの上限に達する可能性がある旨）
- YouTube API による再生リスト作成は行わない（書き込み権限不要・クォータ節約・安全性の観点）
- 生成後、再生用URLを表示
- **YouTubeアプリで開く**:
  - `youtube://` スキームでYouTubeアプリを起動（インストール済みの場合）
  - フォールバックとしてブラウザで開く

## 4. 画面構成

### 4.1 ログイン画面
- Googleログインボタン

### 4.2 メイン画面（ダッシュボード）
- グループ一覧の表示
- グループ選択 → 日付範囲指定 → フィルタ実行のフロー

### 4.3 グループ管理画面
- グループの作成・編集・削除
- チャンネル割り当て（検索付き）

### 4.4 フィルタ結果画面
- 動画一覧（選択可能、投稿日の新しい順）
- 「再生用URLを生成」ボタン
- 生成完了後のリンク表示（50本超の場合は警告付き）

## 5. 非機能要件

### 5.1 API クォータ管理
- YouTube Data API の無料枠: **1日 10,000 ユニット**
- 主なコスト:
  - チャンネル登録一覧取得(subscriptions.list): 1ユニット/リクエスト
  - チャンネル詳細取得(channels.list): 1ユニット/リクエスト（最大50件一括）
  - 動画一覧取得(playlistItems.list): 1ユニット/リクエスト
- **節約戦略**:
  - search.list（100ユニット）の代わりに playlistItems.list（1ユニット）を使用
  - 再生時間（videos.list）は取得しない（表示対象外）
  - アップロード再生リストIDはチャンネル一覧取得時にまとめて取得（channels.listで最大50件一括）
- **1回のフィルタ操作あたりの見積もり**（30チャンネル想定）:
  - チャンネル登録一覧: ~2ユニット（ページネーション含む）
  - チャンネル詳細（uploads再生リストID）: ~1ユニット（50件一括）
  - 各チャンネルの動画取得: ~30ユニット
  - **合計: 約33ユニット/回**（個人利用では十分余裕あり）
- **クォータ上限到達時**: 「本日のAPI上限に達しました」エラーメッセージを表示

### 5.2 レスポンシブデザイン
- iPhone（モバイルファースト）で快適に操作できるUI
- PC でも利用可能

### 5.3 PWA対応（最低限）
- `manifest.json` によるアプリ名・アイコンの設定
- ホーム画面に追加して全画面表示で起動可能
- Service Worker によるオフラインキャッシュは不要（オンライン前提のアプリのため）

### 5.4 エラーハンドリング・操作性
- **APIクォータ上限到達時**: 「本日のAPI上限に達しました」メッセージを表示
- **ネットワークエラー時**: シンプルなエラーメッセージを表示
- **データ再取得**:
  - スマホ: プルトゥリフレッシュ（下方向スワイプ）
  - PC: リフレッシュボタン（両方併設）

### 5.5 認証・データ保持
- **セッション管理**: NextAuth.js の JWTセッション方式（アクセストークン・リフレッシュトークンをJWTに含めてCookieで保持）
- ログアウト時もグループ設定はVercel KVに保持（再ログインで復元）
- OAuthテストモード運用のため、リフレッシュトークンは7日で失効（再ログインで対応）
- **PWA + OAuth**: 初回ログインはブラウザで行う前提（PWAスタンドアロンモードではOAuthリダイレクトに制約があるため、運用で対応）

## 6. 制約事項・注意点

- YouTube Data API の利用には Google Cloud Console でプロジェクト作成・API有効化が必要
- OAuth同意画面の設定が必要（テストモードで運用、7日ごとに再ログインが必要）
- `watch_videos` URLは非公式のため、動画数上限（約50本）や将来的な仕様変更の可能性あり
- Vercel無料枠の制限: サーバーレス関数の実行時間10秒
  - 対策: API Routes経由で1チャンネルずつ取得（フロントから `/api/youtube/videos?channelId=xxx` を3〜5件並列で呼び出し）+ 進捗表示
- Vercel KV無料枠の制限: 1日3,000リクエスト、256MB
  - グループ設定の保存専用に限定すれば十分（動画キャッシュ等には使用しない）

## 7. 将来的な拡張候補（スコープ外）
- キーワードフィルタ
- 再生時間フィルタ
- グループのインポート/エクスポート
- 既存再生リストへの追加
- 複数アカウント対応

## 実装方針

### ディレクトリ構成（概要）
```
TubeMixer/
├── src/
│   ├── app/              # Next.js App Router
│   │   ├── page.tsx      # ダッシュボード
│   │   ├── login/        # ログイン画面
│   │   ├── groups/       # グループ管理画面
│   │   └── api/          # API Routes
│   │       ├── auth/     # NextAuth.js
│   │       └── youtube/  # YouTube API プロキシ
│   ├── components/       # UIコンポーネント
│   ├── lib/              # ユーティリティ
│   │   └── youtube.ts    # YouTube API クライアント
│   └── types/            # 型定義
├── public/               # 静的ファイル・PWAアセット（manifest.json、アイコン）
├── next.config.js
├── tailwind.config.ts
├── tsconfig.json
└── package.json
```

### 検証方法
1. `npm run dev` でローカル起動し、Googleログイン → チャンネル一覧取得を確認
2. グループ作成 → チャンネル割り当て → フィルタ実行 → 動画一覧表示を確認
3. 再生用URL生成 → YouTubeで再生されることを確認
4. iPhoneのSafariからアクセスし、モバイルUIとYouTubeアプリ連携を確認
5. Vercelにデプロイし、本番環境で動作確認
